Home‑Lab Build Notes

  • Sensors: Mirror traffic to a Suricata/Zeek VM; export EVE JSON to SIEM.
  • CTI: Run MISP in Docker; ingest OTX/abuse.ch/ThreatFox; store LLM‑extracted IOCs with context.
  • LLM: Local model (e.g., Llama 3.x) via Ollama or vLLM; RAG over your TI notes; guardrails for IOC regex validation.
  • Rule Pipeline: LLM drafts Suricata/YARA/Sigma → run unit tests on pcaps → human approve → publish to Suricata.
  • SIEM: Elastic/OpenSearch or Wazuh; dashboards for Suricata alerts, anomalies, and IOC matches.
  • Response: Simple playbooks to update blocklists (pf/ipset/iptables) and notify via Slack/Matrix.

Keep the LLM in the loop for explanations and summarization, but hold a human approval gate for actions and rule changes.
